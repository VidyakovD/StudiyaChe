# Политика обработки персональных данных (внутренний документ)

Этот документ — печатная (утверждаемая приказом) копия публичной политики
со страницы [/privacy](../src/app/privacy/page.tsx).

При проверке РКН попросит **именно** утверждённый локальный нормативный акт,
а не страницу сайта (страница — публикация, не первоисточник).

## Как это оформить

1. Скопировать текст с [/privacy](../src/app/privacy/page.tsx) в Word.
2. На первой странице — гриф утверждения:
   ```
                              УТВЕРЖДЕНА
                              приказом ИП Видякова Д.К.
                              от «___» __________ 20__ г. № ___

                              ___________________ /Видяков Д.К./
                              М.П. (при наличии)
   ```
3. Содержание документа — точно совпадает с текстом на сайте.
   Любое изменение публикуется одновременно: в файле и на сайте,
   оформляется новым приказом.
4. Хранить распечатанный экземпляр у Оператора.

## Связанные приказы

- Приказ о назначении ответственного за обработку ПДн — см. [PRIVACY-OFFICER-ORDER.md](./PRIVACY-OFFICER-ORDER.md)
- Приказ об утверждении настоящей Политики
- Приказ об утверждении формы Журнала обращений субъектов ПДн

## Список локальных актов, которые могут запросить при проверке

Для ИП достаточно минимального набора:

| Документ | Где взять |
|---|---|
| Уведомление об обработке ПДн (подача в РКН) | [RKN-NOTIFICATION.md](./RKN-NOTIFICATION.md) |
| Приказ о назначении ответственного | [PRIVACY-OFFICER-ORDER.md](./PRIVACY-OFFICER-ORDER.md) |
| Утверждённая Политика обработки ПДн | этот файл |
| Журнал учёта обращений субъектов ПДн | [PDN-REQUEST-LOG.md](./PDN-REQUEST-LOG.md) |
| Согласия пользователей на обработку ПДн | автоматически фиксируются в БД (`User.consentToProcessingAt`) |
| Выписки из СУБД о составе обрабатываемых данных | по запросу |
